## 段落

段落是最普通、也最常用的写作单位。一段文字不必急着承担太多，它可以只是把一个想法放稳，让读者有地方落脚。

写技术文章时，段落负责解释来龙去脉；写读书笔记时，段落负责留住当时的心绪；写日常时，段落也可以只是接住一个短暂经过的念头。

## 引用

引用适合放一段被特别拎出来的话。它可以来自别人，也可以来自自己写作时临时记下的一句旁白。

### 无出处引用

#### 写法

```markdown
> 有些想法太轻，先放进纸页里，才不至于被风吹散。
>
> **提示**：引用里同样可以使用 _Markdown 语法_。
```

#### 效果

> 有些想法太轻，先放进纸页里，才不至于被风吹散。
>
> **提示**：引用里同样可以使用 _Markdown 语法_。

### 带出处引用

#### 写法

```markdown
> 读书不是为了抵达某个地方，而是为了让路变得更长一点。<br />
> <cite>纸上微光</cite>
```

#### 效果

> 读书不是为了抵达某个地方，而是为了让路变得更长一点。<br />
> <cite>纸上微光</cite>

## 代码块

### 写法

使用三个反引号可以创建代码块。在第一组三个反引号后面写上语言名称，例如 `html`、`css`、`typescript`、`bash`，就可以得到对应的高亮。

````markdown
```html
<!doctype html>
<html lang="zh-CN">
  <head>
    <meta charset="utf-8" />
    <title>Markdown 示例</title>
  </head>
  <body>
    <p>你好，纸上微光。</p>
  </body>
</html>
```
````

### 效果

```html
<!doctype html>
<html lang="zh-CN">
  <head>
    <meta charset="utf-8" />
    <title>Markdown 示例</title>
  </head>
  <body>
    <p>你好，纸上微光。</p>
  </body>
</html>
```
